## What this repo is

M.Eng research project (ECE2500Y, Prof. Deepa Kundur's group, U of T). A 3-tier pipeline:
(1) flow-based IDS on IEC-104 traffic → (2) SHAP attribution → frozen `AnomalyEvent` JSON
contract → (3) LightRAG contextualizer that grounds explanations in a synthetic IEEE 14-bus
grid knowledge base and produces narratives for 3 personas (OT Operator / Data Scientist /
Regulator). Evaluated by CPAS rubric with an independent LLM-as-judge, A/B/C ablation tiers.

**The single most important document is `docs/IMPLEMENTATION_PLAN.md`** — heavily refined
across sessions; it is the source of truth for architecture, phases, agent-executable specs
(Appendix A), and environment pitfalls (Appendix B). Read it before doing anything.
Other key docs: `README.md` (repo map), `docs/data_reality.md` (data limitations note),
`src/ccir/schemas/anomaly_event.py` (the frozen contract).
